Darsning Asosiy Maqsadi

O'quvchilarga tayyor statik HTML shablonni Django backendiga to'g'ri ulashni o'rgatish. Dars oxirida ular nafaqat CSS va rasmlarni ochishni, balki formadan bazaga ma'lumot saqlash va bazadan dinamik ma'lumotlarni HTML ga chiqarishni to'liq tushunishlari shart.

Vaqt 90 daqiqa
Daraja Beginner
Texnologiya Django, Jinja
Natija Zip fayl & DB

Dars O'tish Xaritasi

1-Qadam (Student Slayd 1-2) 10 min

Mashina va Dvigatel Analogiyasi

Darsni boshlashdan oldin, o'quvchilarga hozirgi holatni tushuntiring. Ularning qo'lida tayyor "Frontend" bor, lekin u o'lik holatda.

Qanday tushuntirish kerak:

"Tasavvur qiling, sizga eng so'nggi rusumdagi chiroyli mashina berishdi. Ruli bor, pedallari bor, rangi zo'r (Bu bizning HTML/CSS/JS). Lekin kapotini ochsangiz, ichida dvigateli (motor) yo'q!

Agar Jo'natish tugmasini bossangiz, hech narsa bo'lmaydi. Bugun biz Python va Django orqali shu mashinaga motor qo'yishni va uni harakatlantirishni o'rganamiz!"

2-Qadam (Student Slayd 3-5) 15 min

Fayllarni Joylash va Ruxsatnomalar

O'quvchilar hamma faylni 1 ta papkaga tashlashga o'rganib qolishgan. Djangoning qat'iy qoidalarini (Templates va Static) o'rgating.

templates/

Bu yerga faqat HTML fayllar tushadi. .css yoki rasmlar emas!

static/

Oldingi 'assets' papkasi nomi 'static' ga o'zgaradi va ichida css, js, rasmlar bo'ladi.

Urls.py dagi muhim xato:

Kodni yozishganda urls.py da + static(settings.MEDIA_URL...) qismini unutishadi.

# O'quvchilarga ayting:
"Django xavfsizlik sababli o'zidagi rasmlarni tashqariga, ya'ni brauzerga bermaydi. Ushbu kod orqali biz unga 'Shu rasmlarni ko'rsatishga ruxsat beraman' deymiz."
3-Qadam (Student Slayd 6-8) 20 min

"Sinib qolgan sayt" va Jinja Sehrlari

Loyihani ilk bor run qilganda sayt xunuk, CSS siz ochiladi (6-slayd). Bu ajoyib moment! O'quvchilar qo'rqib ketishganda, ularga Jinja nimaligini tushuntiring.

<link href="assets/css/style.css">

<link href="{% static 'css/style.css' %}">

{% load static %} - bu kalit

HTMLning eng tepasida {% load static %} yozish shart. O'quvchiga ayting: "Bu kalit. Kalit bilan Django omborxonasini (static) ochmasangiz, pastdagi rasmlar va dizaynni ololmaysiz".

4-Qadam (Student Slayd 9-10) 25 min

Formalar va Ma'lumotlar Bazasi (POST)

Endi Frontend orqali foydalanuvchi ma'lumotlarini bazaga saqlaymiz. Bu Backend'ning eng muhim jarayonlaridan biri.

Frontend (HTML) method="POST"
name="ism" {% csrf_token %}
Backend (Views) request.POST.get('ism')
kelgan datani o'qish
Database Model.objects.create(...)
Bazaga saqlash

CSRF Token - Qorovul:

O'quvchilarga ayting: {% csrf_token %} bu sayt qorovulidir. Xakerlar boshqa saytdan turib bizning bazamizga soxta ma'lumot jo'nata olmasligi uchun Django ushbu "maxsus pasport"ni talab qiladi. Pasport bo'lmasa, so'rov rad etiladi (403 xatolik).

  • Xatolarni (syntax error) mustaqil topishga o'rgatadi.
  • Qisqa muddatli xotiradan uzoq muddatliga o'tkazadi.

2-Vazifa: Dinamik Sahifalar (Mentors & Portfolio)

O'quvchilar Mentorlar va Portfolio sahifalarini {% static %} orqali ulagach, u yerdagi ma'lumotlarni to'g'ridan-to'g'ri Admin paneldan keladigan qilishlari (Dinamik) kerak.

Kutilayotgan Model: Mentor
class Mentor(models.Model):
    ism = models.CharField(...)
    soha = models.CharField(...)
    rasm = models.ImageField(...)
{% for mentor in mentors %}
  {{ mentor.ism }}
{% endfor %}
Kutilayotgan Model: Portfolio
class Portfolio(models.Model):
    sarlavha = models.CharField(...)
    muallif = models.CharField(...)
    rasm = models.ImageField(...)
{% for port in portfolios %}
  {{ port.sarlavha }}
{% endfor %}
Mentor uchun muhim tekshiruv!

Tekshirayotganda albatta /admin ga kiring. Agar o'quvchi ma'lumotni Admin paneldan kiritmagan bo'lsa-yu, HTMLda Loop ({% for %}) aylanmasa, sahifada bo'shliq ko'rinadi. Ularga Admin paneldan ma'lumot qo'shishni unutmasliklarini eslating.
