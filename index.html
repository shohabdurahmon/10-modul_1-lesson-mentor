1-Dars: Django Integratsiyasi

Qo'llanma

Dars Maqsadi

Ushbu darsning asosiy maqsadi o'quvchilarga tayyor statik HTML shablonni Django bazasiga ulashni (integratsiya qilishni) o'rgatishdir. Ular shunchaki dizayn emas, balki formalardan (POST) ma'lumot qabul qilishni va datani bazadan chaqirishni tushunib yetishlari kerak.

Vaqt 90 daqiqa
Daraja Beginner
Mavzu Django Static & Jinja
Metod Interaktiv Vizual
1-qadam (10 min)

Kirish: Mashina va Dvigatel anologiyasi

Slayd 2 dagi "Tayyor Frontend"ni ko'rsating. O'quvchilarga bu xuddi chiroyli qilingan mashina (kuzov) ekanligini, lekin uning ichida dvigateli (Backend) yo'qligi uchun yura olmasligini ayting.

Mentorga maslahat:

O'quvchiga savol bering: "Agar mana shu saytdagi Jo'natish tugmasini bossak, ma'lumot hozir qayerga boradi?" Javob: "Hech qayerga, chunki backend yo'q!" Bu ularni integratsiya muhimligiga ishontiradi.

2-qadam (20 min)

Papkalar Strukturasi va Django Qoidalari

O'quvchilar ko'pincha barcha fayllarni bitta joyga tashlab qo'yishga odatlangan. Slayd 3 da Djangoning tartibi haqida gapiring: HTML fayllar faqat templates/ ga, rasmlar va dizayn (CSS) static/ ga tushadi.

STATIC_URL = 'static/' # Brauzer qidiradigan yo'l

STATICFILES_DIRS = [...] # Djangoga kompyuterdagi manzilni ko'rsatish

O'quvchilarda ko'p uchraydigan xato:

urls.py oxiriga + static(settings.MEDIA_URL...) ni qo'shishni esdan chiqarishadi. Ularga buning sababini "Django yopiq ombor, biz maxsus ruxsat yozmasak u rasmlarni tashqariga bermaydi" deb tushuntiring.

3-qadam (15 min)

"Sinib qolgan sayt" va Jinja

Slayd 6 ni ko'rsatganda o'quvchilar asabiylashishi yoki qo'rqishi mumkin ("Sayt nega xunuk bo'lib qoldi?"). Ularni tinchlantiring va Jinja ning kuchini ko'rsating.

Eski usul (Ishlamaydi) <link href="assets/css/style.css">
Djangoda (To'g'ri usul) <link href="{% static 'css/style.css' %}">

"Eng tepadagi {% load static %} - bu o'sha omborni ochish kalitidir. Kalitsiz hech narsa ulab bo'lmaydi!"

4-qadam (25 min)

Formalarni Database'ga ulash (POST)

Endi dvigatelni yoqamiz! Slayd 9 va 10 da formadan kiritilgan Ism va Telefon raqamlar orqadagi ma'lumotlar bazasiga qanday yetib borishini sodda tushuntirib bering.

HTML Form method="POST" {% csrf_token %}
Views.py Ism va Telefonni
ushlab oladi
Database Lead.objects.create(...)

Xavfsizlik tushunchasi:

{% csrf_token %} ni qo'riqchi (Security) ga qiyoslang. Xakerlar saytimizga soxta forma yubormasligi uchun Django doim ushbu "maxsus muhr"ni talab qiladi.

5-qadam (20 min)

Uy Ishini Tushuntirish: Dinamik Ma'lumotlar

Slayd 11 da o'quvchilarga beriladigan uy ishi juda muhim. Ular nafaqat qolgan HTML larni `{% static %}` orqali ulashlari, balki Mentorlar va Portfolio sahifasini Dinamik (Database dan o'qiydigan) qilishlari shart! Mentor sifatida buni vizual doskada tushuntiring:

Vazifa 1: Mentorlar Modeli

O'quvchi models.py da Mentor klasini yaratib, uni HTML dagi kartochkaga ulashi kerak.

Class Mentor(models.Model):
    ism = CharField(...)
    soha = CharField(...)
    rasm = ImageField(...)
{{ mentor.ism }} {{ mentor.soha }}

Vazifa 2: Portfolio Modeli

Huddi shunday, o'quvchilar ishi ham backend orqali aylanib chiqishi tushuntiriladi.

{% for project in portfolios %}
  <img src="{{ project.rasm.url }}">
  <h3>{{ project.sarlavha }}</h3>
{% endfor %}
{{ project.sarlavha }} {{ project.muallif }}

Mentor eslatmasi:

O'quvchilarga uy ishini boshlashdan oldin `Admin Panel` ni sozlash va uning ichidan bir-ikkita ma'lumot (mentor va proyekt) qo'shish kerakligini qattiq tayinlang. Bo'sh bazada tsikl (loop) aylanmaydi!
